## Правила использования в тестах

1. Использовать этот вход только для end-to-end проверки реальной синхронизации (создание записи офлайн → онлайн-синк → появление в облаке → чтение после `page.reload()`/очистки локального кэша).
2. Все тестовые записи, созданные этим пользователем, помечать явным маркером — например, `project: '__SMOKE_TEST__'` — чтобы отличать от реальных производственных данных.
3. После каждой проверки — удалять созданные тестовые записи (мягкое удаление `_deleted:true` по принятому в проекте паттерну, либо через `dbDelete`, если запись не должна попасть в историю).
