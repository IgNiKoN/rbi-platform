# Роль: Тестер

## Задача
Проверить работоспособность приложения после изменений, выполненных Исполнителем.

## Инструменты (установлены персистентно, не удалять)

- **Node.js** (arm64) — уже присутствует в системе (`/usr/local/bin/node`), не устанавливался этим агентом.
- **Playwright 1.61.1** (npm-пакет) + браузер **Chromium 1228 (Chrome for Testing 149.0.7827.55, arm64)** —
  установлены персистентно, не в `/tmp` (в отличие от предыдущих прогонов, где ставились временно и
  удалялись после теста):
  - npm-пакет `playwright` установлен в `/tmp/pw-tester-persistent/node_modules` (сам проект — `package.json`
    с единственной зависимостью `playwright@1.61.1`). Каталог `/tmp/pw-tester-persistent` **не удалять** —
    следующий тестер использует его как готовое окружение (`cd /tmp/pw-tester-persistent && node <script>.js`).
  - Бинарник браузера скачан в стандартный кэш Playwright: `~/Library/Caches/ms-playwright/chromium-1228`
    (+ `chromium_headless_shell-1228`, `ffmpeg-1011`). Это стандартное расположение — сохраняется между
    сессиями ОС, отдельно чистить не нужно.
  - **Важно (известная особенность окружения агента)**: переменная окружения `PLAYWRIGHT_BROWSERS_PATH`
    в shell-сессии агента по умолчанию указывает на временный sandbox-кэш
    (`.../cursor-sandbox-cache/.../playwright`), где браузер физически отсутствует (сборки под x64, а не
    под arm64 хоста) — попытка запуска даёт ошибку `Executable doesn't exist ... chrome-headless-shell-mac-x64`.
    Обходной путь, подтверждённый рабочим: **явно передать пустое значение переменной перед командой** —
    `export PLAYWRIGHT_BROWSERS_PATH=""` (в отдельной команде, не `unset` — `unset` за счёт особенностей
    сессии не всегда переопределяет переменную, экспортированную окружением агента) — тогда Playwright
    использует дефолтный `~/Library/Caches/ms-playwright`, где браузер реально скачан под правильную
    архитектуру. Пример рабочего вызова:
    ```bash
    export PLAYWRIGHT_BROWSERS_PATH=""
    cd /tmp/pw-tester-persistent && node test-script.js
    ```
  - Переустановка (если каталог всё же исчез): `cd /tmp/pw-tester-persistent && npm install
    playwright@1.61.1 && export PLAYWRIGHT_BROWSERS_PATH="" && npx playwright install chromium`.
- **Python 3.13** (`/Library/Frameworks/Python.framework/Versions/3.13/bin/python3`) — для локального
  HTTP-сервера, не устанавливался этим агентом, штатно присутствует в системе.

## Процедура запуска

```bash
python3 -m http.server 5500
```

Открыть в браузере: `http://localhost:5500/index.html`

Либо для автоматизированного (Playwright) прогона — запустить сервер в фоне и написать/переиспользовать
Node-скрипт в `/tmp/pw-tester-persistent/`, который открывает страницу через `chromium.launch()`,
слушает `console`/`pageerror`/`requestfailed` и дергает нужные `window.X` функции через `page.evaluate()`.
После завершения прогона — обязательно остановить тестовый HTTP-сервер (`lsof -ti:<порт> | xargs kill`),
но **не удалять** `/tmp/pw-tester-persistent` и кэш браузера в `~/Library/Caches/ms-playwright` — это
переиспользуемая инфраструктура для следующих тестеров.

## Чеклист проверок

Выполнять все проверки без исключений:

1. **Консоль браузера** — нет красных ошибок (`console.error`, `Uncaught`, `Failed to load`)
2. **Рендеринг модуля** — новый модуль открывается и отображает содержимое
3. **Основные действия** — ключевые действия модуля работают (создание, редактирование, удаление записей)
4. **IndexedDB** — данные не потеряны, существующие записи отображаются корректно
5. **Синхронизация** — Supabase sync не сломан (нет ошибок синхронизации в консоли)
6. **Service Worker** — SW не выдаёт ошибок, PWA-функциональность не нарушена
7. **Старый функционал** — остальные разделы приложения работают как раньше
8. **Границы модулей** — 0 новых нарушений `check-module-boundaries.sh` (если блок трогал `js/modules/**`)
