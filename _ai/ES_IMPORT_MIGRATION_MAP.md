# ES Import Migration Map — аудит 66 classic-файлов перед переводом `<script>` → `<script type="module">`

> Источник задачи: `_ai/ROADMAP.md`, инициатива «Нативные ES import вместо classic-script» (66 файлов, Блок 1/N — этот документ). Только аудит, ни одна строка `.js`/`.html` не менялась.

## 0. Как читать карту

Для каждого файла:
- **Группа** — одна из `services` (36) / `shared` (13) / `core` (6) / `construction/features` (6) / `quality/features-tail` (5); совпадает с ROADMAP 1:1.
- **`<script>`-позиция** — номер строки тега в `index.html` (на момент аудита, до любых будущих правок).
- **Присваивает в `window.*`** — полный перечень имён, которые файл публикует (порядок = порядок первого присвоения в файле).
- **Читает `window.*` других 66-файлов** — только имена, которые: (а) определены (`window.NAME =`) в одном из ЭТИХ ЖЕ 66 файлов, (б) не определены в этом же файле. Общий служебный `window.RBI`/`window.RBI.services.*`/`window.RBI.registry` — сквозной паттерн почти везде (см. §1), в таблицах ниже упоминается только один раз на файл, без раздувания.
- **Top-level immediate-код** — код вне функций/обработчиков, выполняющийся сразу при исполнении скрипта и вызывающий что-то из другого файла. Найдено программной проверкой глубины `{}` (не эвристика на глаз) — см. §2, единственные найдены случаи явно поимённо, для всех остальных файлов — «нет».
- **DOMContentLoaded/load-листенер** — да/нет + что делает, с номером строки.
- **Потребители** — classic-скрипты (в этом же 66-наборе или вне него) и уже-`type="module"` файлы, которые читают хотя бы одно имя, публикуемое файлом (поиск по всему репозиторию, не только по 66-набору).

## 1. Сквозной паттерн: `window.RBI`

27 из 66 файлов (все, кто оборачивает себя в `(function(){ ... window.RBI = window.RBI || {}; ... }())`) как читают, так и пишут `window.RBI` — де-факто общий namespace-объект, а не «зависимость одного конкретного файла от другого». Любой файл этой группы технически «зависит» от того, что `window.RBI` уже существует как объект (или создаёт его сам первым, если выполняется раньше остальных) — это не риск класса 1 (не порядок-зависимый вызов функции), а безопасный merge-паттерн (`window.RBI = window.RBI || {}`), работающий независимо от порядка выполнения. Список файлов, участвующих в этом паттерне (пишут `window.RBI` в первых строках IIFE): `config.service.js`, `sync.service.js`, `permission.service.js`, `object-directory.service.js`, `contractor-directory.service.js`, `storage.service.js`, `settings.service.js`, `session.service.js`, `inspection.service.js`, `report.service.js`, `file.service.js`, `task.service.js`, `sk.service.js`, `game.service.js`, `knowledge.service.js`, `analytics.service.js`, `ai.service.js`, `masterData.service.js`, `template.service.js`, `app-mode.service.js`, `company.service.js`, `user-context.service.js`, `template.utils.js`, `math.utils.js`, `toast.utils.js`, `rbi-core.js`, `app-shell.js`, `app.entry.js`. Не рассматривается далее как риск.

Это не блокер этого блока, а входные данные для проектирования конкретного код-блока перевода `sync`.**
